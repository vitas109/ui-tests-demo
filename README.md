# Galaxy Store 🌌

Интернет-магазин атрибутики по вселенной "Звёздные войны" с UI и скриншот тестами на TypeScript + Playwright и Allure отчётностью.

## 🌐 Деплой

- **Основной сайт:** https://vitas109.github.io/ui-tests-demo/
- **Allure отчёт:** https://vitas109.github.io/ui-tests-demo/allure-report

## 🚀 Возможности

- **Каталог товаров** - 16 товаров (световые мечи, одежда, мерч)
- **Фильтрация и сортировка** - по категории, цене, популярности и остатку
- **Корзина** - добавление/удаление товаров, сохранение в localStorage
- **Отзывы** - возможность оставить отзыв на купленный товар
- **Личный кабинет** - история заказов и отзывов
- **Авторизация** - с хардкодными учётными данными
- **Промокоды** - SW10 даёт скидку 10%
- **Оплата** - тестовый платёж (карта 1111 1111 1111 1111)

## 📋 Технологии

- **Frontend**: HTML, CSS, TypeScript
- **Build**: Vite
- **Linter**: ESlint
- **Testing**: TypeScript + Playwright
- **Test Reporting**: Allure
- **CI/CD**: GitHub Actions

## 🛠️ Установка

```bash

npm install
```

## 🧪 Структура тестов (Page Object Model)

Тесты используют паттерн **Page Object Model (POM)** для лучшей поддержки и читаемости кода.

## ✅ Преимущества Page Object Model

1. **Инкапсуляция** - скрытие деталей реализации UI
2. **Повторное использование** - методы используются в разных тестах
3. **Поддержка** - легче поддерживать при изменениях UI
4. **Читаемость** - тесты становятся более понятными
5. **Масштабируемость** - легко добавлять новые тесты и страницы


### Page Objects

Каждый класс Page Object инкапсулирует взаимодействие с элементами конкретной страницы:

| Класс | Страница | Описание |
|-------|----------|----------|
| `BasePage` | - | Базовый класс с общими методами |
| `HomePage` | Главная | Логотип, hero-секция, карточки категорий |
| `CatalogPage` | Каталог | Товары, фильтры, сортировка |
| `CartPage` | Корзина | Товары, промокоды, оформление |
| `AuthPage` | Авторизация | Модальное окно входа/регистрации |
| `ProfilePage` | Личный кабинет | Профиль, заказы, отзывы |
| `CheckoutPage` | Оплата | Форма заполнения данных и оплата |
| `ResultPage` | Результат | Страницы успеха/отказа |

## 📸 Скриншот тесты

Проект включает **33 скриншот теста** для визуального сравнения UI с эталонными изображениями.

### Что тестируется

| Группа | Количество | Описание |
|--------|------------|----------|
| Главная страница | 5 | Базовый вид, hover-эффекты, мобильная и планшетная версии |
| Каталог | 10 | Фильтры, сортировка, hover на товарах, разные версии |
| Корзина | 5 | Пустая/заполненная, промокоды, изменение количества |
| Оплата | 3 | Форма, успешный/неуспешный платёж |
| Отзывы | 2 | Базовый вид, выбор рейтинга |
| Профиль | 3 | Неавторизованный/авторизованный, с заказами |
| Авторизация | 2 | Модальные окна входа и регистрации |
| Header | 2 | С авторизованным пользователем, с товарами в корзине |
| Footer | 1 | Базовый вид |

### Работа со скриншот тестами

**Первый запуск** (генерация эталонов):
```bash
npm run test -- --grep "Скриншот" --update-snapshots
```

**Последующие запуски** (сравнение):
```bash
npm run test -- --grep "Скриншот"
```

**Обновление эталонов** после изменений UI:
```bash
npm run test -- --grep "Скриншот" --update-snapshots
```

### Настройки сравнения

В тестах используется параметр `maxDiffPixels` для допуска небольших различий:
- `100` — для статических элементов (header, footer)
- `150` — для страниц с динамическим контентом
- `500` — для страниц с динамическими данными (номера заказов, даты)

### Отчёт о различиях

При несовпадении скриншотов Playwright создаёт:
- `*-actual.png` — текущий скриншот
- `*-diff.png` — изображение с подсветкой различий
- `error-context.md` — детали ошибки
